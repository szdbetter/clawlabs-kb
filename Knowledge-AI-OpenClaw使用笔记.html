OpenClaw 使用笔记

核心概念

  • OpenClaw 是 AI Agent 框架，跑在 VPS 上，通过 Discord/Telegram 等渠道交互
  • 支持子智能体 (Sub-agent) 派发任务
  • 内置定时任务 (Cron)、记忆系统 (OpenViking)、多模型调度

开发环境

  • Claude-Code 必须用 PTY 模式，否则会挂起
  • 项目目录：~/.openclaw/projects/
  • 配置文件：/root/.openclaw/openclaw.json

2026-03-21 更新

OpenViking 2.0 升级决策

  • 结论：OV 2.0 可安全替换 OV 1.0，无损 LCM 压缩能力
  • OV 2.0 的 compact 方法会委托给 OpenClaw 内置 LegacyContextEngine（LCM底层引擎）
  • OV 2.0 插件类型变为 context-engine，同时接管记忆+上下文管理两个 slot
  • Docker测试全通过：插件加载✅ 连接OV Server✅ 历史记忆兼容✅
  • 升级路径：① 下载OV 2.0插件到 extensions/openviking/ ② pip upgrade openviking 0.2.5→0.2.9 ③ 配置替换 ④ 重启网关
  • 已确认可执行，等待实施

MiniMax Mini-Agent 评估

  • 本质是 MiniMax 版的 Claude-Code，单模型单会话面向编码
  • 核心能力：文件/Shell工具、Session Note记忆、ACP协议→Zed集成
  • 结论：对我们帮助有限，我们已有 OpenClaw sessions_spawn 体系，ROI不值得折腾
  • 唯一价值：需要纯 M2.7 单模型编码时可用

Fiona 进程异常排查教训

  • 症状：Fiona后台跑了大量 openclaw 进程（7-8个并发）
  • 原因：launchd 多个 plist（gateway + node + guardian watchdog）+ 手动启动进程未清理并存
  • 排查方法：ps aux | grep openclaw | grep -v grep

Context Window 满了的教训

  • 症状：某个 Discord thread 的 bot 静默不响应
  • 原因：context window 满（204800 tokens 上限），该频道大量内容（调研报告+HTML+代码）堆到 655KB
  • 解决：重置 session
  • 预防：高频内容频道用完一个主题后主动重置会话

2026-03-21 更新（VPS运维 & bot-monitor）

LossLess-Claw 压缩策略调优（2026-03-20）

  • 问题：Context 过载，bot 响应慢或不响应
  • 调整：contextThreshold 0.75→0.55，freshTailCount 32→16
  • 效果：context 用到 55% 就开始压缩，只保护最近 16 条不被压缩，有效释放 context 空间
  • 操作：通过 gateway config.patch 热更新，无需停服

bot-monitor.py 检测盲区教训（2026-03-21）

  • 症状：Reese 超时时，bot-monitor 完全没报警
  • 根因：bot-monitor 只匹配了 FailoverError / lane task error / discord inbound worker timed out 三种正则，Reese 超时的日志格式不在其中
  • 另一问题：告警阈值是单 channel ≥5 次才报，低频超时被忽略
  • 建议：补充 timeout / 429 / 连接断开等模式，或改成检测「N 分钟内无响应」方式

VPS 进程管理最佳实践（2026-03-21）

  • claude 孤儿进程：SSH 断开后 Claude Code 进程变孤儿驻留内存，kill 不影响会话历史（保存在 ~/.claude/），安全清理
  • github-backup 从 cron 迁移到 systemd timer：cron 方式频繁 timeout，改用 systemd timer 更稳定
  • 旧：OpenClaw cron job（5f001d6f）每2h执行，经常超时
  • 新：openclaw-git-backup.timer systemd timer 每1h执行，稳定运行
  • 命令：systemctl enable --now openclaw-git-backup.timer
  • Fiona 架构说明：Fiona = 企微 bot，只处理企业微信频道；VPS Reese = Discord bot，处理 Discord。两者不共享 channel，排查时注意区分
