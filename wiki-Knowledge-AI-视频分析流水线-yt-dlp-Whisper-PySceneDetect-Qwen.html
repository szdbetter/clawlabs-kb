视频分析流水线：yt-dlp → Whisper → PySceneDetect → Qwen2.5VL

在 M3 Max (128GB) 上验证通过的完整视频深度分析工具链。 首次测试用例：1.5小时 YouTube 喜剧小品合集，产出 6.9KB 结构化分析报告。

核心结论

  1. 全流程本地化可行：575MB 视频在 M3 Max 上 40-50 分钟完成全链路分析，无需 API 调用
  2. 多维度交叉分析：语音转写 + 视觉场景检测 + LLM 综合推理，比单一模态分析深入数倍
  3. 产出丰富：一次分析产出 2759 行对白、1093 个场景、1346 张关键帧、6.9KB 结构化报告
  4. 可复制性强：工具链全开源，流程标准化，适合批量分析

工具链架构

YouTube URL
  │
  ├─ yt-dlp ──────────── 下载视频 (webm/mp4)     ~5min
  │
  ├─ ffmpeg ──────────── 提取音频 (wav)           ~2min
  │
  ├─ faster-whisper ──── 语音转写 (JSON/SRT)      ~15-20min
  │   └─ CPU int8 量化，模型：tiny/base/small
  │
  ├─ PySceneDetect ───── 场景切割 + CSV 报告      ~10min
  │   ├─ 关键帧提取 (keyframes/)
  │   └─ 场景截图 (scenes/, 每场景3张)
  │
  └─ Qwen2.5VL ────────── 视觉理解 + 综合分析     LLM 推理
      └─ 输入：转写文本 + 关键帧 + 场景数据
      └─ 输出：结构化分析报告

各工具职责

工具 版本/模型 输入 输出 耗时(1.5h视频)
yt-dlp 最新 YouTube URL webm/mp4 ~5min
ffmpeg 系统版 视频文件 WAV 音频 ~2min
faster-whisper base/int8 WAV 音频 JSON(segments+text) ~15-20min
PySceneDetect 最新 视频文件 CSV + 帧图片 ~10min
Qwen2.5VL vl-max 文本+图片 Markdown 报告 LLM 依赖

分析报告结构（已验证模板）

通过首次测试验证的报告输出结构：

1. 内容结构分析

  • 整体架构识别（多段式/单线/多线）
  • 每段的核心设定、冲突、笑点技巧

2. 节奏分析（量化）

  • 总场景数、平均场景时长、切换频率
  • 对白密度（行/分钟）
  • 节奏特征总结

3. 台词风格分析

  • 语言特色（句长、口语化程度）
  • 金句摘录
  • 幽默/叙事技巧分类

4. 视觉特征分析

  • 基于关键帧的风格识别
  • 镜头运用模式

5. 爆款规律总结

  • 可复制的方法论提炼
  • 对短剧制作的启示（结构、节奏、台词、叙事、视觉）

环境要求

硬件（已验证）

  • Apple M3 Max, 128GB RAM
  • SSD 至少需要 ~2GB 临时空间（视频+音频+帧）

软件依赖

# Python 3.14 + video-tools-env 虚拟环境
pip install yt-dlp faster-whisper scenedetect[opencv]
# 系统依赖
brew install ffmpeg  # macOS

存储规划

  • 原始视频和帧数据保留在本地（reese），不上传服务器
  • 分析报告（KB级）可同步到知识库
  • 关键帧可按需上传（用于 AI 视觉分析）

应用场景

  1. 竞品分析：分析竞品短剧/视频的节奏、叙事技巧、视觉风格
  2. 选题验证：分析爆款视频提取可复制的成功要素
  3. 剧本优化：量化分析自有作品的节奏和对白密度，对标爆款
  4. 素材库建设：从视频中提取关键帧作为参考图/分镜素材
