飞书 Bot API 权限限制与排障

来源：[飞书] muse-feishu 会话 714eee17 + 0d7a9903 入库时间：2026-04-09

核心内容

图片下载 234008 错误（平台级限制）

错误码: 234008 — "The app is not the resource sender"

根因: 飞书图片下载 API 的平台级限制 — Bot 只能下载自己发出的消息中的图片，无法下载群内其他用户发的图片。这不是权限配置问题，是飞书的设计限制。

变通方案: 1. 让用户把图片通过 DM 私聊发给 Bot 2. 用文字描述替代图片传递 3. 让用户发送飞书文档链接（文档内的图片 Bot 可以访问）

跨组织文档访问

不同飞书租户间的用户无法直接访问文档（即使有链接），报 400 错误。

解决方案: 通过 API 设置文档为公开链接访问：

PATCH /open-apis/drive/v1/permissions/{token}/public
Body: {
  share_entity: "anyone",
  link_share_entity: "anyone_readable",
  security_entity: "anyone_can_view",
  comment_entity: "anyone_can_view"
}

注意: share_entity 的合法值为 anyone | same_tenant | only_full_access，不是 anyone_can_view（后者用于其他字段）。

Token 获取

POST /open-apis/auth/v3/tenant_access_token/internal/
Body: { app_id, app_secret }
返回: { tenant_access_token, expire: ~3278s }

关键教训

  1. 飞书 Bot 图片权限是平台级限制，不要在配置上浪费时间排查
  2. 跨组织协作必须设置公开链接，API 字段命名不直觉（share_entity vs security_entity 含义不同）
  3. 多 Bot 实例通过不同 App ID 实现角色分离，共享同一个 OpenClaw 网关
