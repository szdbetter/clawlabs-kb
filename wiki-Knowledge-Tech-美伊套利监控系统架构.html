美伊套利监控系统架构

背景

2026年3月，美伊战争期间，Polymarket 上存在大量美伊相关预测市场（ceasefire/hormuz/military等）。项目目标：构建一套套利机会识别系统，通过程序化分析盘口定价的逻辑矛盾和单调性违反来发现alpha。

系统架构

Dashboard (port 1980)
├── server.js（Express后端）
│   ├── GET /api/iran-markets      → Polymarket Gamma API数据聚合
│   ├── GET /api/iran-arbitrage    → 纯计算，套利机会分析
│   └── GET /api/iran-rules        → Claude API语义解读（带1h缓存）
│
├── public/iran-arb.html（前端）
│   ├── Section 1: 全盘口一览表（按事件组分组）
│   ├── Section 2: 套利机会扫描（红色高亮卡片）
│   ├── Section 3: LLM规则解读（手风琴折叠面板）
│   └── Section 4: 信息源状态（IMF PortWatch / Exa API）
│
└── 自动刷新：每30秒

后端API设计

/api/iran-markets

数据源：https://gamma-api.polymarket.com/events?active=true&closed=false（无需Auth）

过滤条件：title/description 包含 iran/ceasefire/hormuz/military

返回结构：

{
  "events": [{
    "slug": "...",
    "title": "...",
    "markets": [{
      "question": "...",
      "yes_price": 0.115,
      "no_price": 0.885,
      "volume": 50000,
      "liquidity": 10000,
      "end_date": "2026-03-31"
    }]
  }]
}

/api/iran-arbitrage

套利识别逻辑（三种类型）：

1. 单调性检验（跨日期同系列）

同一事件组（如ceasefire）按截止日期排序：
- Mar31停战概率 < Apr30停战概率 < May31停战概率（应递增）
- 若Mar31(11.5%) > Apr30(8%) → 违反单调性 → 套利机会

输出示例：

{
  "type": "monotonicity_violation",
  "market_a": {"slug": "...", "date": "Mar31", "yes": 0.115},
  "market_b": {"slug": "...", "date": "Apr30", "yes": 0.08},
  "edge": 0.035,
  "action": "BUY Apr30 YES, SELL Mar31 YES",
  "confidence": "HIGH"
}

2. 概率一致性检验

  • "宣布结束军事行动"概率 应 >= "停战协议签署"概率（宽松条件 ≥ 严格条件）
  • "美军进入伊朗领土"概率 与 "停战"概率 应负相关
  • 违反以上逻辑关系 → inconsistency_score 高 → 套利机会

3. 价差套利

逻辑矛盾 + 价差足够大（edge > 阈值）→ 具体操作建议

/api/iran-rules

使用 Claude API 对每个盘口进行语义解读，提取： - trigger_conditions：触发Yes的精确条件 - gray_areas：歧义点/边缘案例 - logical_relation：与其他盘口的逻辑关系 - resolution_risk：官方认定标准的解析风险

缓存策略：结果缓存到 /tmp/iran_rules_cache.json，超过1小时才刷新

API Key读取（无dotenv）：

const envContent = fs.readFileSync('/root/.openclaw/.env', 'utf8');
const apiKey = envContent.match(/ANTHROPIC_API_KEY=(.+)/)?.[1]?.trim();

BABA研报生成流程（扩展场景）

同一套dashboard-mvp基础设施也支持BABA研报生成：

  1. 拉取 BABA 最新财报数据
  2. Claude 生成中文研报文本（Markdown格式）
  3. 写入 /root/.openclaw/workspace/projects/digital-twin-advisor/content/
  4. 可推送到微信公众号（push_draft_with_images.py）

相关文件： - projects/digital-twin-advisor/content/kazike_boss_wechat_article_v2.md - projects/digital-twin-advisor/content/payload_with_images.json - projects/digital-twin-advisor/content/push_draft_with_images.py

关键决策

  1. 纯计算套利引擎：套利分析不调外部API，纯基于市场数据计算，延迟极低
  2. 前端自动刷新30秒：盘口价格实时性要求高，但不需要WebSocket，30秒轮询足够
  3. LLM解读缓存1小时：盘口规则不会频繁变化，缓存大幅降低API成本
  4. Express静态托管：前端直接放public/目录，无需单独前端构建流程
  5. nodemon热重载：开发时server用nodemon，修改文件自动重载，无需手动重启

教训

  1. Polymarket API完全开放：无需注册API Key，gamma-api.polymarket.com直接请求即可
  2. 单调性检验是最可靠的套利信号：纯数学推理，不依赖主观判断
  3. LLM适合做语义桥梁：把盘口英文description翻译成结构化trigger_conditions，是LLM的最强用途
  4. 大盘口 ≠ 好流动性：volume高但liquidity低的盘口，实际无法执行套利
  5. 时区问题：Polymarket end_date需要统一转换为UTC，否则跨日期比较出错

    server_name clawlabs.top;

    location /elon      { ... }
    location /report    { ... }
    location /iran      { ... }

  3. 扩展新路径 — 只需在 nginx 加 location /xxx 块，无需改代码
