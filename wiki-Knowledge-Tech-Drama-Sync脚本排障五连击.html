Drama Sync 脚本排障五连击

来源：[Discord] drama 频道 2026-04-09 实战排障 频道ID：1488677061093494966 入库时间：2026-04-09

背景

drama_site_sync.sh 是短剧系统的核心同步脚本，负责将 Agent 写入的文件自动同步到 SQLite 数据库。一次全面审计发现了 5 个 bug，均属于"看上去能跑但数据错位"的隐蔽类型。

Bug 清单与修复

Bug 1：03_script 同步为 step "4"（应为 "3"）

现象：网站剧本页面空白，API 查不到 step=3 数据 根因：sync_file "$slug" "4" "$section" "$f" 把剧本文件标记成了分集的步骤号 修复：改为 sync_file "$slug" "3"

Bug 2：04_episodes 同步为 step "3"（应为 "4"）

现象：分集数据在 step=3 位置，与剧本冲突 根因：与 Bug 1 互为镜像——两个目录的步骤号写反了 修复：改为 sync_file "$slug" "4"

Bug 3：os.walk 遍历了 thumbs/ 子目录

现象：asset library 场景标签下混入角色缩略图，总数翻倍（26 vs 13） 根因：os.walk(img_dir) 递归进入 thumbs/ 子目录，thumbs 不在 CATEGORY_MAP 中默认归类为 scene 修复：dirs[:] = [d for d in dirs if d != 'thumbs'] 剪枝

Bug 4：缺少 catalog.json 优先级

现象：项目手动维护的 catalog.json 被自动生成覆盖 根因：脚本无条件执行自动扫描，忽略已有 catalog 修复：检测 catalog.json 存在时直接使用，跳过自动生成

Bug 5：outline images 只在目录不存在时复制

现象：大纲阶段新增的概念图不会同步到 05_images 根因：if <span class="wikilink-missing" title="未找到笔记"> ! -d "$target_img_dir" </span> 条件只在首次创建时执行 修复：改为 always mkdir -p + cp -n（no-clobber 增量复制）

通用教训

  1. 映射错误是 shell 脚本最隐蔽的 bug：目录名→步骤号的映射纯靠人工对齐，没有编译器帮你检查
  2. os.walk 默认递归所有子目录：任何非内容目录（thumbs, cache, temp）都需要显式剪枝
  3. "默认分类"陷阱：当分类逻辑有 fallback/default 时，未覆盖的目录名会静默归入默认类别
  4. 文件优先级原则：用户手动维护的文件应优先于自动生成，自动化不应覆盖人工精调
