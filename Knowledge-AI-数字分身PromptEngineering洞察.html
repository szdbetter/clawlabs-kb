数字分身 Prompt Engineering 关键洞察

来源：Claude-Code / projects/digital-twin-advisor/cases/liurun/ 记录日期：2026-03-23

stage: evergreen

核心结论

System Prompt 迭代是数字分身质量的关键杠杆，通过5轮迭代可将分数从17/50提升至50/50。

stage: evergreen

五轮迭代经验

版本 分数 核心问题
v1 17/50 模型输出元回答，不进入角色
v2 19/50 工作区上下文污染
v3 41/50 字数超标（1800-2300字）
v4 39/50 字数约束导致不稳定
v5 50/50 全部问题解决 ✅

stage: evergreen

关键发现

1. 上下文污染（最危险）

Claude CLI 会读取工作区目录文件，导致模型误以为自己在处理软件项目，而非扮演KOL数字分身角色。 解法：Prompt中明确加入"忽略所有其他上下文"。

2. 元语言禁止（效果最显著）

禁止这些词会大幅提升角色沉浸度： - "我会..." - "我来帮你..." - "你想讨论什么..." - "好的，我来..."

解法：在Prompt中明确列出禁止词列表。

3. 格式硬约束

  • 字数范围：800-1500字（不能只写上限）
  • "不要有任何开场白，直接开始第一段故事"
  • 禁止超过3点的列举

4. 三段式结构

故事 → 规律 → 金句是刘润风格的核心模式，必须显式定义在Prompt中。

stage: evergreen

通用SOP（适用所有数字分身）

  1. 基线测试：先生成v1，跑10道测试题取得基线分数
  2. 问题归类：分析失败原因（上下文污染/元语言/格式/角色跳出）
  3. 单变量迭代：每次只改一个维度
  4. 禁止词列表：每个分身都需要定制禁止词
  5. 格式刚性：字数范围比字数上限更重要

stage: evergreen
