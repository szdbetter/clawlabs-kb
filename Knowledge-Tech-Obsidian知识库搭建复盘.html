Obsidian 知识库搭建复盘

背景

老板想建立个人知识库，能自动整理日常对话、语音、研究成果，并在知识之间产生关联。

关键决策

1. 工具选型：Obsidian

  • 本地优先，纯 Markdown 文件，数据完全自主
  • 双向链接 [[]] 是核心能力，笔记多了形成知识图谱
  • 不需要账号登录，没有数据库，文件夹就是 Vault

2. 同步方案：iCloud + GitHub（不用 Obsidian Sync）

  • 设备间同步：iCloud 管 Apple 设备间（免费）
  • 接入 OpenClaw：GitHub 私有仓库 szdbetter/obsidian-vault
  • VPS 每小时自动 push → Mac git clone 到 iCloud 目录 → iPhone 自动看到
  • iPhone 不支持 obsidian-git 插件，需要 Mac 做中转

3. 存储架构：VPS 为主

  • Vault 路径：/root/.openclaw/obsidian-vault/（最初在 ~/obsidian-vault/，后迁入 .openclaw/）
  • 不需要专门的 skill，直接读写 .md 文件即可
  • 安装了 obsidian-direct 技能包提供搜索便利（可选）

4. 知识库目录结构

Inbox/          → 自动捕获的原始素材
Knowledge/      → 整理后的知识卡片（AI/Crypto/Business/Tech）
Projects/       → 项目相关
Daily/          → 每日汇总
MOC/            → Map of Content 索引导航页

自动化配置

  • ⏰ 每小时同步：自动 git push 到 GitHub
  • ⏰ 每晚23:00知识归档：回顾当天对话 → 提炼知识卡片 → 分类写入 → 自动关联 → push
  • 📢 写入通知规则：每次写入必须告知 ✍️ [标题] 已写入知识库，目录：xxx

踩坑记录

  1. 内置 skill 不适用 — OpenClaw 自带的 obsidian skill 依赖 obsidian-cli（brew），Linux VPS 装不了
  2. 其他频道找不到知识库 — 需要把路径写入长期记忆，所有会话才能定位
  3. iPhone 不支持 obsidian-git — 必须通过 Mac 中转（iCloud + git）
  4. Obsidian 不需要登录 — 登录入口是付费 Sync 服务，我们不用

核心认知

  • Obsidian 是容器，OpenClaw 是往里面填内容和整理内容的引擎
  • 单用 Obsidian 要自己写自己整理；接上 AI 后知识库自动生长
  • 知识之间的 <span class="wikilink-missing" title="未找到笔记">双向链接</span> 是 Obsidian 最大价值，笔记量大了才能体现
