MiniMax AI 内容生成工作流

更新日期：2026-03-22

stage: evergreen

一、概览

MiniMax 是 ClawLabs 的主力 AI 内容生成工具，覆盖： - 视频生成（text_to_video / image_to_video） - 音乐生成（music_generation） - 图片生成（text_to_image）

所有调用通过 mcporter 工具（MiniMax MCP Server）完成。

stage: evergreen

二、视频生成实践

电商 Vlog 生成 SOP（liujun 项目，2026-03-21~22）

需求：给服装品牌生成真实感手机竖屏 Vlog，展示多件产品

关键提示词要素： - 竖屏 9:16，720×1280 - 手机实拍质感（噪点、手持抖动、无美颜） - 真实场景（办公室/公寓走廊），非影棚打光 - 产品图片参考：/root/.openclaw/workspace/projects/e-commerce/liujun/pic - 多产品：在视频中转场切换

批量提交方式： 1. 每件产品单独生成一个 ~6 秒视频片段（image_to_video） 2. 使用 ffmpeg 拼接 + fade 转场，合成完整视频 3. MCP server 每次生成后可能崩溃，需要重启后继续

ffmpeg 拼接命令：

# 生成带 fade 转场的拼接视频
ffmpeg -i seg1.mp4 -i seg2.mp4 ... \
  -filter_complex "xfade=fade:duration=0.5:offset=5.5..." \
  -c:v libvpx-vp9 -b:v 400k output.webm

输出规范： - ⚠️ 所有生成产物 严禁 写入 workspace - 统一输出到 /tmp/<项目名>/，如 /tmp/liujun-vlog/ - 需要发给用户时，从 /tmp/ 路径上传

发布至 Discord

  • WebM 格式可直接在 Discord 预览
  • 超过 8MB 需先上传到临时 CDN（如 catbox.moe）再发链接

stage: evergreen

三、音乐生成实践

Kimi 成长记系列歌曲（2026-03-21）

模型：music-2.0（MiniMax music_generation）

注意事项： - 单次生成上限约 90 秒，需分段生成再拼接 - 超时阈值需设置：MCPORTER_CALL_TIMEOUT=300000（5分钟） - 拼接用 ffmpeg：-filter_complex "[0:a][1:a]acrossfade=d=3[out]"

风格描述关键词： - 周深风格：空灵男高音、电子弦乐交织、钢琴为主 - 风格参考：三角洲星海入梦（游戏OST风格） - ❌ 用户不喜欢：主旋律/史诗感/春晚风格 - ✅ 用户喜欢：现代 C-Pop 流行、吉他+钢琴+轻电子

完整版制作流程： 1. Part 1 生成（~90秒）→ Part 2 生成（~90秒） 2. ffmpeg crossfade 拼接（3秒淡入淡出） 3. 末尾加淡出处理

stage: evergreen

四、ClawLabs VI 规范（内容生成配色）

确立日期：2026-03-20（内容事业部thread）

配色方案

颜色角色 Hex 值 用途
主色（VI红） #D2231E 关键元素高亮
深红 #A00F0F 次要强调
亮红 #EB5041 悬停/点击状态
背景 #141416 深色底
文字 #FFFFFF 主文字

图片生成原则

  • 主调：米黄 + 墨绿 + 金色（复古学术风）
  • VI红点缀：仅作关键元素高亮（占画面 3-5%），不做主色调
  • 视觉节奏：安静-突出-安静
  • 公众号排版：标题/小标题使用 VI 红色

stage: evergreen

五、小红书内容发布流程

确立日期：2026-03-20-22（内容事业部thread）

为什么不能一键发布

  • 小红书没有公开官方 API
  • Twitter Basic 发帖 API 需要 $100/月
  • 现有方案都需要 cookie，有封号风险

当前最优发布方式

  1. 生成 GitHub Pages 发布页（如 szdbetter.github.io/nvidia-5layer-report/xhs_publish.html）
  2. 页面提供：
  3. 📋 一键复制全文按钮（文字进剪贴板）
  4. 图片支持长按直接保存（CDN 直链）
  5. 手机端：打开链接 → 复制文字 → 保存图片 → 小红书 App 粘贴发布

小红书字数限制

  • 正文 ≤ 1000 字（超出需裁剪）
  • 精简后反而信息密度更高，更适合传播

小红书 MCP（待激活，2026-03-22）

  • 已通过 Docker 安装 MCP Server
  • 阻塞项：需要登录 cookie（VPS IP 被小红书风控）
  • 激活步骤：
  • 用户在浏览器登录 xiaohongshu.com
  • 安装 Cookie-Editor 插件 → Export → Header String
  • 发给 Reese 注入 MCP 配置

stage: evergreen

六、教训总结

  1. 生成产物统一 /tmp：workspace 不能堆媒体文件，会污染 git 备份
  2. MCP server 不稳定：每次生成后可能崩溃，需要容错处理
  3. 90 秒上限：音乐生成分段是刚需，不是 bug
  4. 小红书 IP 封锁：VPS IP 在小红书是黑名单，必须用本地 cookie
  5. Discord 上传限制：超过 8MB 用 CDN 中转，不要直传
