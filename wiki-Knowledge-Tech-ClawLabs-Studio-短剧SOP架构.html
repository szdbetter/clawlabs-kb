ClawLabs Studio 短剧 SOP 架构

来源：[Discord] drama 频道 2026-04-09 对话沉淀 频道ID：1488677061093494966 入库时间：2026-04-09

核心内容

六步流水线（v5.0，自动建站）

Step 1 调研 → Step 2 大纲 → Step 3 剧本 → Step 4 分集 → Step 5 出图 → Step 6 分镜

每一步按标准目录结构存储，自动建站（零 deploy）。

数据架构

Agent 写内容 → API 写入 SQLite → 即时生效
用户访问 → Nginx → 统一模板 → API 动态加载
图片 → 直接从项目目录读取

关键决策：移除"建站"步骤，改为全自动同步。Muse Agent 只管写文件，sync 脚本管其他一切。

目录结构

projects/drama/{slug}/
├── 01_research/        # Step 1 调研
│   ├── SOURCES.md      # 原始素材（用户提供的文字/图片/链接）
│   ├── NOTES.md        # 调研分析（竞品+风格+市场+知识库对照）
│   └── BRIEF.md        # 项目简报
├── 02_outline/         # Step 2 大纲
│   ├── outline.md      # 文学叙事式大纲（1500-2000字）
│   └── images/         # 概念图（角色出场图+氛围图）
├── 03_script/          # Step 3 剧本
│   ├── EP01-EP05.md    # 每批5集一个文件
│   └── ...
├── 04_episodes/        # Step 4 分集
│   ├── episode-guide.md
│   └── EP01.md ...
├── 05_images/          # Step 5 出图
│   ├── catalog.json    # 图库索引
│   ├── characters/     # CHAR-{NAME}-{NNN}.png
│   ├── scenes/         # SCENE-{LOCATION}-{NNN}.png
│   ├── props/          # PROP-{ITEM}-{NNN}.png
│   └── fx/             # FX-{DESC}-{NNN}.png
└── 06_storyboard/      # Step 6 分镜
    └── EP01.md ...

零 Deploy 架构演进

旧架构：每个项目复制一份 index.html + deploy 脚本手动部署 新架构： - Nginx 将所有 /drama/{slug}/ 请求指向同一个模板文件 - 模板从 URL 提取 slug，从 API 动态加载数据 - 图片通过 Nginx alias 直接从项目源目录读取 - 安全迁移：先建 /drama-v2/ 测试路由，验证通过后切换

关键教训： 1. SOP 中必须规定 Markdown 标题级别（## EP01：标题），否则不同 Agent 自由发挥导致解析失败 2. 排版规范必须全局通用（加粗=蓝色，加粗斜体=琥珀色，斜体=灰色） 3. 每次功能修改后自动跑 /QA 验证 4. deploy 脚本内嵌的 heredoc 模板极易与动态模板不同步 — 应只维护一份模板

调研步骤标准化

  • Part 1 SOURCES.md：只放用户提供的原始素材（文字、图片、链接），不含分析
  • Part 2 NOTES.md：竞品分析+风格参考+市场分析+知识库对照+创作方向建议（必须真实搜索）
  • Part 3 BRIEF.md：一句话概念+核心设定+结构设计+差异化优势
