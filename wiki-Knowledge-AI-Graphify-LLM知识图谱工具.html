Graphify：把 Karpathy 的 LLM Wiki 从理念变成了产品

type/summary #domain/ai

一个命令，把任何文件夹变成可查询的知识图谱。

核心洞察

Karpathy-LLM-Wiki-方法论 提出了「用 LLM 编译个人知识库」的理念，但本质是「LLM 维护 Flat File Wiki」。Graphify 把它升级为「LLM + AST 共同构建 Knowledge Graph」——从文档管理到图计算的范式提升。

Graphify（⭐ 3.2k Stars）是第一个将 Karpathy 理念工程化实现的开源项目，发布一周即获得广泛关注。它作为 Claude-Code、Codex、OpenCode 的 Skill 运行，输入 /graphify 即可将文件夹中的代码、文档、论文、截图构建为可查询的知识图谱。

七级处理流水线

Graphify 的核心是一条七级流水线，每级为独立 Python 模块，通过纯 dict 和 NetworkX 图通讯，无共享状态：

  1. 检测 — 扫描文件夹，识别文件类型
  2. 提取 — 双轨并行提取（见下节）
  3. 构建 — 将提取的实体和关系构建为 NetworkX 图
  4. 聚类 — Leiden 算法基于边密度进行社区检测
  5. 分析 — 发现 God 节点、意外连接等模式
  6. 报告 — 生成 GRAPH_REPORT.md
  7. 导出 — 输出 graph.json + graph.html 交互式图谱

双轨提取引擎

这是流水线中最精妙的设计：

  • 左轨：tree-sitter AST 静态解析 — 对代码文件进行确定性语法树分析（类、函数、import、调用图）。零 Token 消耗，毫秒级完成，支持 16 种语言
  • 右轨：Claude 语义提取 — 对文档、论文、图片启动并行子代理（每 20-25 文件一批），利用视觉能力提取概念、实体、关系和设计决策的「为什么」

信任审计链

每条边附带置信度标签，实现知识的可追溯性：

标签 含义 置信度
🟢 EXTRACTED 直接来源于代码/引用 1.0
🟡 INFERRED 合理推论（共享结构等） 0.4-0.9
🔴 AMBIGUOUS 不确定，需人工审查 0.1-0.3

特殊节点类型 rationale_for：提取代码中的 # WHY: / # HACK: / # NOTE: 注释，记录不只是做了什么，还有为什么。

71.5x Token 压缩

  • 首次运行：消耗 Token 进行提取和图谱构建（一次性成本）
  • 后续查询：读取紧凑的 graph.json 而非原始文件，节省 71.5x Token
  • SHA256 缓存：增量更新只处理变更文件
  • 本质：付一次「编译」成本，获得无限次高效查询

Karpathy 理念 vs Graphify 实现

维度 Karpathy Graphify
存储格式 Markdown Wiki NetworkX 图 + JSON
知识发现 LLM 手动维护反向链接 Leiden 算法自动社区检测
代码处理 全部走 LLM 代码走 AST（零 Token），文档走 LLM
可信度 无标注 三级置信度标签 + 分数
增量更新 概念性描述 SHA256 缓存 + --update + git hooks
多模态 提及图片 完整视觉理解（截图/图表/白板）
工具生态 Obsidian HTML + Obsidian + Neo4j + MCP + SVG

关键设计决策

  • 无向量数据库：聚类完全基于图拓扑结构，图结构本身就是相似性信号
  • 超边（Hyperedges）：支持 3 个以上节点参与同一概念/流程
  • 全模态支持：代码、PDF、Markdown、截图、架构图、白板照片
  • Always-On 模式：Claude Code 的 PreToolUse 钩子在每次搜索前先读取图谱
  • MCP 服务器模式：--mcp 暴露为 MCP stdio 服务器，提供 query_graph、get_neighbors、shortest_path 工具

快速上手

pip install graphifyy && graphify install
/graphify .                              # 一键运行
/graphify ./raw --mode deep              # 深度模式
/graphify ./raw --update                 # 增量更新
/graphify query "概念之间有什么联系？"      # 查询图谱
