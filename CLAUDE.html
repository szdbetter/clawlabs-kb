仓库结构

obsidian-vault/
├── Inbox/          # 快速收集箱，未处理的原始素材
├── Knowledge/      # 结构化知识卡片
│   ├── AI/         # AI工具、框架、模型
│   ├── Tech/       # 技术架构、系统设计、自动化
│   ├── Crypto/     # 加密货币、链上套利、预测市场
│   ├── 投资/       # 股票、投资方法论
│   └── Business/   # 电商、内容创业、商业模式
├── Concepts/       # 概念/实体页（跨文章知识节点）
├── MOC/            # Map of Content，导航索引层
├── Projects/       # 项目跟踪与任务看板
│   └── TODO.md     # 统一任务看板
└── Daily/          # 自动生成的日报（由 cron 驱动）

知识组织规范

三层架构

  1. 原始文章层（Knowledge/）：具体场景的深度记录，包含实操细节
  2. 概念实体层（Concepts/）：跨文章的核心概念聚合页，每个概念一个文件
  3. 导航层（MOC/）：领域级索引，按主题聚合概念页

双向链接规范

  • 文章提到核心概念时，必须用 <span class="wikilink-missing" title="未找到笔记">ConceptName</span> 链接到 Concepts/ 下的对应页
  • 概念页维护"引用此概念的文章"列表
  • 同领域笔记之间也应建立 <span class="wikilink-missing" title="未找到笔记">Note Title</span> 横向链接

笔记标签体系

  • #stage/evergreen：已完善的长期笔记
  • #stage/draft：草稿状态
  • #domain/ai、#domain/crypto、#domain/business：领域标签
  • #type/concept：概念页
  • #type/moc：导航页

常用 AI 操作命令

/process-inbox

扫描 Inbox/ 目录，对每个文件： 1. 判断归属领域 2. 提取关键概念，添加 <span class="wikilink-missing" title="未找到笔记">Concept</span> 双向链接 3. 移动到对应 Knowledge/ 子目录 4. 在相关 Concepts/ 页中添加引用

/update-concepts

扫描所有 Knowledge/ 笔记，提取高频概念： 1. 检查 Concepts/ 是否已有对应页面 2. 新概念：创建概念页（含定义、引用文章列表） 3. 已有概念：更新引用文章列表

/weekly-digest

生成本周知识沉淀周报： 1. 扫描本周修改的 Knowledge/ 文件 2. 按领域分组，提取核心洞察 3. 输出到 Daily/ 目录

/rebuild-moc [domain]

重建指定领域的 MOC 页面： 1. 扫描该领域所有 Knowledge/ 和 Concepts/ 文件 2. 按子主题分组 3. 生成带摘要的导航目录
