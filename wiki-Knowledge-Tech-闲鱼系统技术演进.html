闲鱼电影票系统技术演进

背景

闲鱼电影票自动回复系统，用于卖家自动处理买家询价、图片识别选座截图、自动报价和订单管理。

架构演进：CDP → WebSocket

旧架构（CDP方案）问题

  • 图片识别失败：fetchImgBase64 的 URL key 匹配 bug，截图方案找不到目标元素
  • 架构脆弱：DOM scraping 依赖 class 名，闲鱼前端一更新就挂
  • 30秒轮询：不是实时推送，响应延迟高
  • 依赖 Fiona Mac + Chrome + CDP，运维复杂

新架构（WebSocket方案）

VPS单进程
├── FastAPI server.py（主进程）
│   ├── 复用：analyze_intent() / calc_price() / 订单逻辑 / Discord通知
│   └── startup → asyncio.create_task(ws_client.start())
└── XianyuWSClient（asyncio task）
    ├── 连接 wss://wss-goofish.dingtalk.com/
    ├── 心跳循环（每N秒发 {"lwp": "/!"}）
    ├── Token刷新循环（每3600s）
    └── 消息循环 → 解密 → ACK → on_buyer_message回调

参考项目：XianyuAutoAgent（⭐5.9k），已逆向闲鱼 IM WebSocket 协议

核心技术决策

1. WebSocket协议细节

连接目标：wss://wss-goofish.dingtalk.com/

连接Headers：

headers = {
    "Cookie": cookies_str,
    "Host": "wss-goofish.dingtalk.com",
    "Origin": "https://www.goofish.com",
    "User-Agent": "Mozilla/5.0 ...",
}

注册消息格式：

{
    "lwp": "/reg",
    "headers": {
        "app-key": "444e9908a51d1cb236a27862abc769c9",
        "token": access_token,
        "did": device_id,
    }
}

签名算法：MD5(token + "&" + timestamp + "&" + app_key + "&" + data) - HTTP API app_key: "34839810" - WebSocket app_key: "444e9908a51d1cb236a27862abc769c9"

2. 消息解密流程

Base64 → MessagePack → JSON

核心工具函数（移植自 XianyuAutoAgent/utils/xianyu_utils.py）：

def generate_sign(token, t, app_key, data) -> str
def decrypt(data: str) -> dict        # Base64→MessagePack→JSON
def generate_mid() -> str
def generate_uuid() -> str
def generate_device_id(user_id: str) -> str
def trans_cookies(cookie_str: str) -> dict

3. 图片消息处理（关键难点）

XianyuAutoAgent 原版不支持图片消息（Issue #14/#48 未解决），需自行补全。

探测方案：收到 [图片] 消息时，将完整解密后的 payload 写入 /tmp/xianyu_img_payload.json 调试。

图片URL预期位置： - image.url 或 picUrl 字段 - 或在 syncPushPackage 二级解密数据中 - 最坏情况：用 messageId 调 HTTP API 拉消息详情

图片下载：需要带 Referer: https://www.goofish.com/，CDN图片URL通常带签名token

4. Cookie管理

  • 来源：从 Fiona Mac Chrome F12 手动复制
  • Token刷新：每3600s自动刷新 accessToken（延长Cookie有效期至数天）
  • 过期监控：WebSocket断连超过3次 → Discord告警

5. 单进程集成方案

@app.on_event("startup")
async def startup():
    await init_db()
    cookies = os.getenv("XIANYU_COOKIES_STR", "")
    if cookies:
        ws_client = XianyuWSClient(cookies, on_buyer_message)
        asyncio.create_task(ws_client.start())

关键：server.py 现有逻辑（analyze_intent、calc_price、订单、Discord通知）0改动，全部直接复用。

风险与缓解

风险 缓解措施
图片URL在加密payload中找不到 用messageId调HTTP API拉消息详情；抓包APP对比
Cookie过期快 Token自动刷新延长有效期；Discord过期告警
闲鱼风控封号 回复延迟3-8秒、限频、人工接管模式
图片CDN鉴权 WS协议内图片URL带签名token，比DOM里的src可靠

教训

  1. DOM scraping是反模式：任何依赖前端class名的方案都是定时炸弹，闲鱼前端一更新即崩溃
  2. 逆向协议 > 模拟浏览器：WebSocket实时消息协议比CDP更稳定、延迟更低
  3. 单进程 > 多进程：asyncio task嵌入FastAPI startup，省去进程间通信
  4. MVP先跑通文字，再做图片：图片解析是最高风险点，应先验证基础连通性
  5. Cookie刷新机制是稳定性关键：不加token刷新的方案只能用几小时
