KOL数字分身项目

来源：Discord Thread #KOL数字分身计划 (1481616043016327293, 223条) 提取日期：2026-03-20

一、项目概述

核心理念

不是做"像某某说话"的玩具，而是建立可调用的顾问资产。

当我遇到一个问题时，我希望能立刻切到某个人的思维模型里，听听"他会怎么判断"。

未来最强的人，不一定是自己什么都懂的人，而是手里有一整套顾问系统的人。

二、技术方案

数字分身创建SOP（5步流程）

  1. 锁协议 — 确认语料范围、语言、截止日期
  2. 转语料 — 整理原始素材为raw.md
  3. 抽画像 — 生成advisor_profile.md（思维模型、判断逻辑、表达风格）
  4. 生成Prompt — 生成avatar_prompt.md（用于激活分身的系统提示词）
  5. 强制收口 — 四条件全部满足才算完成

强制收口四条件

只有同时满足以下4条，才允许宣布"已完成"：
1. 三件套生成（raw.md + advisor_profile.md + avatar_prompt.md）
2. 写入 projects/digital-twin-advisor/REGISTRY.md
3. 写入/补强长期记忆（OpenViking）
4. 更新 STATUS.md（已完成/执行中/待确认/已废弃）

缺任意一条 = 未完成，不能叫"完成了"。

三件套文件说明

  • *_raw.md — 原始语料（公开内容、访谈、文章等）
  • *_advisor_profile.md — 顾问画像（专业领域、判断框架、思维特点）
  • *_avatar_prompt.md — 激活提示词（用于在AI对话中召唤该顾问）

四、跨Channel共享机制

现状（截至2026-03-12）

  • 已实现：长期记忆共享（OpenViking是共享记忆库）
  • 已实现：本地项目文件共享（workspace是全局工作区）
  • 已实现：协议已定义（创建流程/三件套/登记注册/状态板）
  • 未完全实现：所有Channel的Agent都自动强制收口

正确调用方式

任何Channel直接用顾问名即可激活分身： - "刘小排，分析一下这个AI产品值不值得做" - "卡兹克，判断一下这个新模型是不是噱头"

五、公众号文章SOP

文章：《我不再"用"AI了，我开始批量制造顾问了》

  • 人设：老板第一人称
  • 风格：卡兹克式，观点鲜明、简单、密度高
  • 标题：选A — "我不再'用'AI了，我开始批量制造顾问了"

核心观点： 1. 做出来 ≠ 完成，必须收口完成才算完成 2. 单个AI不如顾问团：给不同问题匹配不同顾问 3. 所有工作都会变成内容：先干活再提炼内容 4. 批量制造顾问：SOP标准化 = 规模化

公众号发布链路

  1. 文章定稿（Markdown）
  2. 使用字流API格式化（段落间距1.1倍，重要观点加粗/红色）
  3. 生成全中文科技风配图（豆包生图，每个二级标题前一张）
  4. 上传封面到微信素材库（拿thumb_media_id）
  5. 推送草稿箱
  6. 更新CONTENT_REGISTRY

六、已踩的坑与教训

SOP不完整导致的问题

卡兹克案例：三件套文件生成了，但没有写入REGISTRY和长期记忆，导致跨Channel查询时"找不到"。

根因：旧SOP把"三件套生成"当成完成，缺少强制最后一步登记。

修复：把完成态从"协议"升级为"硬门槛"——没有写registry+memory+status，禁止回复"已完成创建"。

微信草稿箱配图问题

  • 只传封面图（thumb_media_id）不等于正文有配图
  • 需要用微信正文图专用上传链路，在HTML正文里嵌入<img>标签
  • 字流API中文需要UTF-8，字段用content而非markdown

七、智囊团模式

价值定位

不是替代思考，而是把高质量思考变成可调度的系统。

未来最强的商业模式： - 不是自己什么都懂的人 - 而是手里有一整套顾问系统： - 一个人看产品 - 一个人看传播 - 一个人看工具 - 一个人看结构化落地 - 汇总成更像真实世界决策的答案

标准化的意义

一旦把流程跑顺了，拥有的不是某一个顾问，而是一条批量生产顾问的流水线。
