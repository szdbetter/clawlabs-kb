爆款视频分析方法论 V0.2

来源：[Discord] 搜索频道 2026-04-08 视频分析实战沉淀 频道ID：1472858439591002211 入库时间：2026-04-09

核心内容

标准化报告模板 V0.2（7 Part + 视频身份卡）

由导演、编剧、短视频运营三方专家评审后升级。

Part 内容 核心产出
Part 0 视频身份卡 基础数据+赛道定位+一句话结论
Part 1 内容结构 叙事架构图（几幕/每段时长/转折点）
Part 2 数据+流量分析 赞播比/互动率/播放效率/标题策略
Part 3 视听制作（导演视角） 镜头语言/声音设计/表演调度
Part 4 喜剧机制（编剧视角） 笑点类型分类/冲突结构/"铺-转-抖"结构
Part 5 执行包 可填空的公式模板/角色模板/Shorts切片策略
Part 6 方法论沉淀 从分析中提炼可复制的规则（→ 自动写入 wiki）
Part 7 待验证假设 未证实的推测 + 跨平台建议

video-analyze Skill 架构

触发：分析视频 / 视频分析 / 爆款分析 / analyze video

流程：
1. SSH reese → pipeline.sh <url> → 下载+转写+场景检测（M3 Max 执行）
2. YouTube Data API v3 → 播放数据（播放量/赞/评论数/发布日期）
3. LLM → V0.2 标准化报告（7 Part）
4. 存储 → reese 本地 + /digest 到 Obsidian vault

5 大爆款规律（跨视频共性）

  1. 多段式结构降低弃看率 — 每段 8-15 分钟独立成章
  2. 密集笑点保持注意力 — 平均每 15-30 秒一个笑点
  3. 情感弧度完整 — 建立设定→冲突→升级→反转→情感落地
  4. 角色群像+化学反应 — 固定演员班底，互相吐槽是核心吸引力
  5. 题材跨度大但调性统一 — 古装/科幻/现代多元，幽默风格一致

关键节奏参数（可复制）

  • 镜头：3-5 秒/场景
  • 对白：25-35 行/分钟
  • 笑点：每 30 秒一个
  • 转折：至少 2-3 层
  • 最后段落：情感升华

Pipeline 踩坑记录

  1. grep -P macOS 不支持 → 改用 sed
  2. yt-dlp --merge-output-format webm 导致音视频分离 → 改用 mp4
  3. Whisper 子进程找不到 ffmpeg → export PATH=/opt/homebrew/bin:$PATH
  4. Whisper --language auto 参数无效 → 不指定或指定具体语言如 --language zh
