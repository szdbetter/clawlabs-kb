[!abstract] 核心洞察 协议逆向 > 模拟浏览器：每一次"从CDP迁移到WebSocket"都是同一个教训——直接对接底层协议，比依赖前端DOM稳定100倍。

技术架构 #自动化 #设计模式 #CDP #WebSocket #轮询 #事件驱动

一、自动化三条路径：CDP vs WebSocket vs API

路径对比

路径 代表项目 优势 劣势
CDP（Chrome DevTools Protocol） 闲鱼电影票自动化项目 v1 零API申请，复用登录态 依赖浏览器进程，前端更新即崩溃
WebSocket（逆向协议） 闲鱼系统技术演进 v2 实时推送，稳定，低延迟 需逆向协议，图片消息处理复杂
官方API 理想情况 最稳定，有文档 闲鱼对个人开发者不开放

闲鱼的迁移路径

CDP方案（旧）→ WebSocket方案（新）
原因：
1. DOM scraping依赖class名 = 定时炸弹，闲鱼前端一更新即崩
2. 图片识别通过fetchImgBase64有URL key匹配bug
3. 30秒轮询 = 响应延迟，用户体验差
4. 依赖Fiona Mac + Chrome物理在线，运维成本极高

迁移收益：
- 实时消息推送（WebSocket心跳循环）
- VPS单进程运行，脱离物理节点依赖
- 协议稳定：wss://wss-goofish.dingtalk.com/ 比DOM可靠

核心教训：DOM scraping是反模式。任何依赖前端class名的方案都是定时炸弹。 → 详见 闲鱼系统技术演进

二、轮询 vs 事件驱动

两种模式对比

场景 方案 频率 适用条件
闲鱼消息（旧） 30秒轮询 0.033Hz 低时效要求，快速MVP
闲鱼消息（新） WebSocket事件驱动 毫秒级 需要实时响应，协议可逆向
美伊套利监控 30秒前端自动刷新 0.033Hz 盘口变化缓慢，30秒足够
SmartMoney链上监控 1小时轮询 0.000277Hz Cloudflare限制，需Playwright规避

选型原则

实时性要求高 + 协议可逆向 → WebSocket事件驱动
实时性要求中 + 有官方API → API轮询（合理频率）
有Cloudflare防护 → Playwright headed模式（住宅IP节点）
数据变化缓慢 → 降频轮询，节省资源

→ SmartMoney监控用Fiona节点（住宅IP）规避Cloudflare，1小时/次：SmartMoney-Fiona监控系统 → 美伊套利用30秒前端轮询已满足需求：美伊套利监控系统架构

三、OCR + 自动化的极限：悟空抢码5秒方案

架构设计

VPS fiona服务端(port 1980)
├── wukong-monitor.js
│   ├── fetchInviteImageUrl()   ← 检测新码
│   ├── ocrInviteCode()         ← Claude Haiku Vision（1-1.5秒）
│   └── pollCheck()             ← 客户端查询接口

Mac客户端 wukong-grab.sh
└── 每2秒 curl /api/wukong/poll
    └── 检测到新码 → AppleScript → 钉钉自动输入

总时序：轮询0-2s + API 0.3s + OCR 1-1.5s + AppleScript 0.5s = 2-4.5s ✅

关键设计决策

  • OCR用Haiku级别：Vision能力足够提取邀请码，价格是Opus的1/20
  • AppleScript是Mac自动化银弹：零依赖，能控制任何有界面的应用
  • 纯bash客户端：Mac端脚本零依赖，比Node.js更易分发
  • 服务端OCR + 客户端执行：计算在VPS，操作在Mac，架构清晰

四、Dashboard架构模式

统一监控面板模式

ClawLabs所有项目监控都收归同一个Dashboard（port 1980）：

dashboard-mvp (Express.js, port 1980)
├── /api/iran-markets      ← 美伊盘口数据（Polymarket Gamma API）
├── /api/iran-arbitrage    ← 套利机会扫描（纯计算，无外部依赖）
├── /api/iran-rules        ← LLM规则解读（带1h缓存）
├── /api/runtime           ← 系统运行状态（cron/OpenViking/daemon）
└── public/
    ├── iran-arb.html      ← 美伊套利监控页面
    └── macro.html         ← 霍尔木兹宏观引擎

两个监控系统对比

系统 数据源 刷新策略 LLM调用
美伊套利监控 Polymarket Gamma API 30秒前端轮询 规则解读，1h缓存
SmartMoney信号 GMGN（Playwright采集） 1小时批量 SM评分引擎

共同模式： 1. 纯计算逻辑（套利/SM评分）不调LLM，延迟最低 2. LLM只做"语义桥梁"（规则解读/意图分析），结果缓存 3. 前端自动刷新 + Discord告警双通道

→ 详见 美伊套利监控系统架构、SmartMoney-Fiona监控系统

五、跨项目技术决策模式总结

模式一：协议逆向 > DOM Scraping

DOM scraping → 协议逆向（WebSocket/API）
稳定性：定时炸弹 → 长期可用
案例：闲鱼CDP → WebSocket迁移

模式二：单进程 > 多进程

asyncio task嵌入FastAPI startup
省去进程间通信
案例：xianyu_ws.py作为asyncio task嵌入server.py

模式三：LLM缓存 > 每次调用

LLM解读结果缓存1小时
避免每次刷新都调API
案例：/api/iran-rules缓存到/tmp/iran_rules_cache.json

模式四：分层架构（物理节点 + VPS）

Fiona（住宅IP）→ 绕Cloudflare采集
VPS → 计算/存储/Dashboard
原则：让物理节点跑脚本，让VPS跑服务
