Claude Code 编程实践笔记

背景

记录使用 Claude-Code 进行自动化开发的关键实践，包括配置方法、实际项目案例和踩坑记录。

一、Claude Code 配置

Dangerous模式（绕过所有权限确认）

Claude Code 默认每次工具调用都要确认，--dangerously-skip-permissions 模式可跳过所有确认。

配置方法一：命令行启动

claude --dangerously-skip-permissions

配置方法二：settings.json（推荐，持久化）

文件位置：~/.claude/settings.json 或项目级 .claude/settings.json

{
  "dangerouslySkipPermissions": true,
  "model": "claude-opus-4-6"
}

配置方法三：OpenClaw启动参数

--permission-mode bypassPermissions

常用启动方式

# 非交互模式（适合自动化/subagent）
claude --print --permission-mode bypassPermissions "任务描述"

# PTY模式（Codex/OpenCode需要）
# 通过exec工具: pty=true

模型选择建议

场景 推荐模型
复杂规划/架构 claude-opus-4-6
日常编码 claude-sonnet-4-6
快速OCR/Vision claude-haiku-4-5-20251001
廉价数据处理 claude-haiku-4-5-20251001

注意：claude-sonnet-4-6-thinking 在某些环境下不可用，会报 "model does not exist" 错误，降级使用 claude-sonnet-4-6。

二、悟空邀请码5秒抢码方案

场景

钉钉悟空邀请码发放后需要在5秒内完成：检测→OCR→自动输入到钉钉桌面客户端。

整体架构（服务端+Mac客户端）

<a href="wiki-Concepts-Fiona.html" class="wikilink">Fiona</a>服务端(port 1980)          Mac客户端
┌─────────────────────┐        ┌──────────────────────┐
│ wukong-monitor.js   │        │ wukong-grab.sh       │
│  ├── fetchInviteImageUrl()  │◄──│  每2秒 curl /api/wukong/poll│
│  ├── ocrInviteCode()        │   │  检测到新码 → AppleScript │
│  └── pollCheck()    │        │  自动输入钉钉+回车    │
│                     │        └──────────────────────┘
│ server.js           │
│  └── POST /api/wukong/poll │
└─────────────────────┘

时序（< 5秒）

轮询间隔 0-2s + API请求 0.3s + Anthropic Vision OCR 1-1.5s + AppleScript 0.5s = 2-4.5s

OCR实现（服务端，Node.js）

// wukong-monitor.js
async function ocrInviteCode(imageUrl) {
  // 使用 Anthropic Vision API，用 https 模块直接调（不加依赖）
  // 模型: claude-haiku-4-5-20251001（快+便宜，Vision足够）
  // prompt: "提取图片中的邀请码，只返回邀请码文本，无则返回空"
}

async function pollCheck(clientUrlId) {
  const { imageUrl, urlId } = await fetchInviteImageUrl();
  if (urlId !== clientUrlId && !state.cachedOcr) {
    const inviteCode = await ocrInviteCode(imageUrl);
    return { changed: true, imageUrl, urlId, inviteCode };
  }
  return { changed: false, imageUrl, urlId, inviteCode: state.cachedOcr };
}

Mac客户端脚本（纯bash，零依赖）

#!/bin/bash
# wukong-grab.sh
LAST_URL_ID=""
while true; do
  RESP=$(curl -s -X POST http://fiona:1980/api/wukong/poll \
    -H 'Content-Type: application/json' \
    -d "{\"lastUrlId\":\"$LAST_URL_ID\"}")

  CHANGED=$(echo $RESP | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('changed',''))")
  CODE=$(echo $RESP | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('inviteCode',''))")

  if [ "$CHANGED" = "True" ] && [ -n "$CODE" ]; then
    # AppleScript自动输入
    osascript <<EOF
tell application "DingTalk" to activate
delay 0.3
tell application "System Events"
  keystroke "a" using command down
  keystroke "$CODE"
  delay 0.2
  keystroke return
end tell
EOF
    afplay /System/Library/Sounds/Glass.aiff
    echo "✅ 抢码成功: $CODE"
    exit 0
  fi
  sleep 2
done

四、常见问题与解决方案

Agent子进程模型不可用

现象：Claude Code 内部 Agent subagent 报 "There's an issue with the selected model (claude-sonnet-4-6-thinking)"

原因：旧版本Claude Code使用thinking模型作为默认子agent模型，但该模型在该环境不可用

解决：直接使用 Bash/Read 工具替代 Agent 子进程；或更新Claude Code版本

大文件读取策略

# JSONL文件（每行一个JSON对象）分批读取
# 优先读前500行（含计划/架构信息）+ 最后200行（含结果/验证）
offset=1, limit=500
offset=end-200, limit=200

httpx vs requests（Python异步场景）

  • asyncio环境必须用 httpx.AsyncClient，不能用 requests（同步阻塞）
  • aiosqlite 对应 sqlite3 的异步版本

教训

  1. Node.js调Claude API不需要SDK：直接用 https 模块，避免依赖问题
  2. AppleScript是Mac自动化银弹：零依赖，能控制任何有界面的应用
  3. OCR选轻量模型：haiku级别的Vision能力足够提取邀请码，价格是opus的1/20
  4. 缓存减少API调用：LLM解读结果缓存1小时，避免每次刷新都调API
  5. 纯bash客户端：Mac端脚本用纯bash，比Node.js更零依赖、更易分发

🆕 2026-03-28 踩坑：订阅账号 Claude Code 403 报错

[来源: Claude Code session 84bce0fc | Fiona节点开发]

问题：在 Fiona Mac 上用 Claude Code，登录后提示 API Error: 403 {"error":{"type":"forbidden","message":"Request not allowed"}}

根因：Claude Code 默认走 API Key 模式，而订阅账号（Max/Pro）无 API Key，直接走 OAuth 鉴权，两者不兼容。

解决方案：

# 用 OAuth 登录而非 API Key
claude auth login
# 或检查是否误设了 ANTHROPIC_API_KEY 环境变量（订阅账号应留空）
echo $ANTHROPIC_API_KEY  # 应为空

补充发现： - Claude Code v2.1.79 在 Mac 订阅模式下未内置 Computer Use，需配合 claude GUI 或单独申请 Computer Use Beta - Computer Use 在 Fiona 本机执行最可靠，通过 VPS 远程控制会因网络延迟和分辨率问题降低稳定性

🆕 2026-03-28 新场景：Fiona节点 Computer Use 滑块处理方案

[来源: Claude Code session 84bce0fc | 闲鱼bot迁移]

背景：闲鱼自动化bot在VPS远程执行时遇到滑块验证码，无法通过脚本模拟。

决策： - 不在VPS侧硬解滑块（反检测成本高） - 在 Fiona 本机通过 Claude Computer Use 临时接管 → 识别+点击滑块 - 正常流程仍走 VPS → Fiona CDP，异常触发再切到 Computer Use
